API Packages

The Digital Script API is organized into packages, similar to Java. However, unlike Java, inheritance is not possible from these classes or packages when creating a script. You can only use the properties and methods of these classes in your scripts.

The TopLevel package is the default package in this API. It is similar to java.lang in Java and does not need to be imported into scripts. It provides standard ECMAScript classes and extensions, such as Error, Date, Function, String, Math, Number, and XML.

The TopLevel.global class contains many common constants and properties used in scripts. Some of these properties include customer, request, and session.

In these packages, there are many classes ending with Mgr. These classes, such as dw.catalog.ProductMgr, retrieve instances of business objects related to the package they belong to. For example, use ProductMgr.getProduct(String id) to get a product using a unique identifier. The method returns a Product instance, which you can use to find information about the product. This pattern is repeated for all managers.
