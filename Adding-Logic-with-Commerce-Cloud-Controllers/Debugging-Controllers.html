Debugging Controllers

One of the most important tools a developer should have is a debugger. It simplifies your workflow by allowing you to pause the execution of your code at specific lines, analyze variable values, and execute your code line by line to ensure it performs as intended.

The Prophet extension in Visual Studio Code not only allows us to sync our code with the sandbox but also provides debugging features. To use it, follow these steps:

  1. Open any controller file and click on the left side of the line number where you have code. A red dot will appear, indicating that a breakpoint is set. If the function is called, execution will pause at this line when the debugger is running.
  2. On the left panel, click on the bug icon.
  3. If you havenâ€™t created a debug configuration before, you will see three options: Debug with Node.js, Run with Node.js, and Create a launch.json file. Choose to create the JSON file.
  4. It will generate a file named launch.json inside a folder named .vscode. Delete its content and replace it with the following:
{
    "version": "0.1.0",
    "configurations": [
        {
            "type": "prophet",
            "request": "launch",
            "name": "Attach to Sandbox"
        }
    ]
}

To start debugging, click the play button on the left side of the dropdown list.
