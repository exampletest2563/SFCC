What are req, res, and next?

Before we move on, let's briefly explain these three parameters that are ALWAYS passed to our functions in controllers. Each step of the middleware chain is a function that takes three arguments: req, res, and next, in this specific order.

  • req stands for request, and it contains information about the server request that initiated the call. It includes user input information and parses query string parameters, assigning them to the req.querystring object.
  • res stands for response, and it contains functionality for outputting data back to the client.
  • next notifies the server that you are done with a middleware step, so it can execute the next step in the chain.

Extending Your Controller

Now that you know how to structure your controller, let’s discuss a feature in SFRA that allows us to extend these controllers.

When I mention "extend," you might wonder why we need a feature for that. Can’t we just modify the controllers’ code directly? If so, you should review the previous module where we explained why this approach can be problematic.

Remember the server import we used for get and post endpoints? It turns out it has other functions we can use to extend existing functions.

By "extending," we mean you can modify the behavior of a controller without altering the original file. To achieve this, you need to create a new cartridge and, within that cartridge, create a new controller.
