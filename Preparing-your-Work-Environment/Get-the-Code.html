Get the Code

There are three ways to download the code from GitHub: using the command line, GitHub Desktop, or downloading the source manually.

To get the code:

  1. Open the GitHub repository.
  2. Click on the Clone or Download button.
  3. Choose Download ZIP.
  4. Save the ZIP file locally.
  5. Unzip the file.

Visual Studio Code

Now let’s get your IDE set up so we can start working. First, install Visual Studio Code.

Install Prophet Extension

  1. Open Visual Studio Code.
  2. Click the menu View > Extensions.
  3. Type Prophet.
  4. Click on the Prophet Debugger result item.
  5. Install it.
  6. Click on the View menu item at the top menu bar and type Prophet in the input field that appears.

Now that Prophet is installed, you can use its commands by pressing CTRL + SHIFT + P or by clicking on the menus View > Command Palette and typing Prophet in the input field that appears.

Open Your Code Using Visual Studio Code

  1. Open Visual Studio Code.
  2. Click on the menu File > Open Folder.
  3. Select the root folder containing all the source code you downloaded.

Install Your Packages

When working with Node.js projects (like SFRA), you need to install all dependencies listed in your package.json file. To do that:

  1. Open the Terminal/Command Line.
  2. Ensure the current directory is the project root folder.
  3. Type: npm install
  4. Press ENTER.
  5. Wait for all the packages to be installed.

Compile Your Code

In your package.json file, you will find several scripts for different purposes such as testing, linters, compiling scripts, uploading, and watchers. To compile your code (as of the time of writing this course), you should run the build script, which compiles your JavaScript, SCSS, and font files. You can run the script directly in VS Code or type npm run build in the terminal.

If you’re editing client cartridge files (like JavaScript or SCSS), you must run this command repeatedly to compile the files and see your changes in the storefront. To avoid manually running the command each time, you can use the watch command. This command recompiles the necessary files whenever you save changes. Type npm run watch.

Create a Server Connection

The final configuration needed is to create a connection between your local code and your sandbox so that your changes are uploaded to the sandbox and implemented on your storefront site.

To set up this connection, you need to create a configuration file containing your credentials and sandbox information. This file should be placed in your root folder, where Prophet will read it to sync with your sandbox and upload your code. Note that Prophet only supports one-way push connections; you cannot pull code from the sandbox to your local machine.

Assuming you have your project open in VS Code, create a file named dw.json (more details can be found in the official documentation).

  1. Fill out your sandbox URL (ignore the part https:// and everything after .com).
  2. Fill out your username.
  3. Fill out your password.
  4. Fill out the code version you are using.
  5. Save the file.
  6. Open the menu View > Terminal.
  7. In the dropdown list, select Prophet Uploader (this allows you to see Prophet’s actions).
  8. Open the menu View > Command Palette (CTRL + SHIFT + P).
  9. Type Prophet.
  10. Select Clean Project/Upload All.

Once the upload is complete, Prophet will start watching for changes. From this point on, any change you make to any file will be uploaded to your sandbox. Your project is now connected to your sandbox, and you will use this connection to upload all your cartridge files to the sandbox.
