Adding Custom Cartridges

There are hundreds of cartridges available that can save time and effort, so you don‚Äôt have to reinvent the wheel. Often, your project will benefit from using these cartridges.

SFCC provides a list of more than 300 cartridges that you can use. To incorporate a cartridge into your project:

  1. Download the source from GitHub.
  2. Copy the cartridge from inside the cartridges folder.
  3. Place it inside the cartridges folder in your project root directory.
  4. Add the cartridge name to your path.

When a call is made to a file, your sandbox searches for the file starting with the first cartridge listed in the cartridge path. For example, if a call is made to the productDetails.isml file and that file exists in two cartridges listed in the cartridge path, the first one found will be used.

Using WebDAV to View Your Cartridges on Business Manager

Administrators and developers use WebDAV to manage files and folders on B2C Commerce servers. If you are unsure whether your file or modifications have been uploaded to your sandbox, you can use this tool to verify.

Follow these steps to ensure that all your cartridges are correctly uploaded to the sandbox:

  1. Log into the Business Manager instance where you want to view cartridge contents (e.g., staging instance).
  2. Select Administration > Site Development > Development Setup.
  3. In the WebDAV Access section, click the Cartridges link.
  4. An authentication dialog might appear if this is your first time accessing it after opening your browser.
  5. Enter your Business Manager username and password.
  6. Click OK.
  7. Click the code version you created when you imported the RefArch site to view the uploaded cartridges.
  8. Click the link corresponding to the name of the cartridge you just created.
