Creating an SFRA Cartridge

When you need to add new functionality to your storefront site, the default approach is to create a new cartridge in your project. The base cartridge, app_storefront_base, contains all the code needed for an SFRA storefront to function in Commerce Cloud Digital. It includes:

  • The business layer, which encompasses all of the server-side components (such as Digital scripts).
  • The simple presentation layer, including ISML templates, common CSS files, forms, and resource files.
  • The specific SCSS, JavaScript, and advanced UI elements required to create the look and feel of the SFRA storefront.

There are two ways to create a new cartridge: manually or using the sgmf-scripts node package. During this training, we will show you how to create it manually since it is a straightforward process.

To create a new cartridge, you essentially need two mandatory files:

  • .project
  • <cartridge_name>.properties

Open your app_storefront_base cartridge and check the file named .project. You will see it looks like this:

Create a new folder inside the cartridge folder and name it, for example, app_storefront_custom. Make a copy of the .project file you opened inside app_storefront_base and paste it into your new folder.

Your new cartridge needs an identical .project file, but you should change the content of the name tag to the name of your new cartridge.

Follow the same process for the .properties file. Check the one in app_storefront_base, copy it to the correct location, and replace all occurrences of app_storefront_base with the name of your new cartridge. At the end, it should look like this:

Whenever you create a new cartridge to add functionality, it is crucial that you maintain the same folder structure that you see in the app_storefront_base cartridge. Otherwise, things may not work correctly.
