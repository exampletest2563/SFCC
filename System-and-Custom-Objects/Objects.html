Objects

Have you ever wondered how Business Manager stores Products, Catalogs, or Categories? How does the API interact with these objects?

Previously, you worked with Controllers and called classes from the SFCC API to retrieve data from Business Manager. The objects you manipulate are actually stored in Business Manager as tables.

System Objects

The first type of object that can be saved into Business Manager is called a System Object. You might wonder why they are called System Objects. The answer is simple: they come pre-configured out-of-the-box when you receive your sandbox.

System Object Types

Let's take a look at the system objects provided out-of-the-box:

  • Go to Administration > Development > System Objects Types

Do you recognize the items pointed to by the green arrows? They are some of the system objects we have been working with in previous chapters.

Check System Object Fields

  • Go to Administration > Development > System Objects Types
  • Click on the Product item
  • Click on the Attribute Definitions tab
  • At the bottom, click Show All

As you can see, the Product Object has almost 100 fields! But as you might recall, we did not see that many fields when editing products in earlier modules. So, what is going on?

Although objects can have many fields, to avoid cluttering the screen, we can limit which fields are displayed when creating or editing an instance of that object. If you go back to check any Product, you will notice that not even half of the fields we just saw are listed there.

Check Which Fields Are Displayed in the Edit Page

  • Go to Administration > Development > System Objects Types
  • Click on the Product item
  • Click on the Attribute Grouping tab
  • Check the names in the Name column: In the Attributes column, you will see a number per row. That number indicates the number of fields inside that grouping.
  • Click on the number related to Search Engine Optimization Support: You will see four fields listed: Page Title, Page Description, Page Keywords, and Page URL.
  • Now, open any product and check if these same fields are listed under the Search Engine Optimization Support set.

As you can see, this approach allows us to customize object pages to display the fields we want and hide those we consider unnecessary for the project.

One last important detail: as mentioned at the beginning of this module, System Objects come pre-configured with your Business Manager. This means you cannot create new System Object Types. If you need a new type of object, you should explore Custom Objects.
