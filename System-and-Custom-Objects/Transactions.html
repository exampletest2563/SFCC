Transactions

When working with objects, whether System or Custom, it's crucial to understand the concept of transactions. You can retrieve objects as much as needed, but creating, editing, or deleting an object involves a different process.

A transaction provides a context for performing atomic changes to persistent business objects. Before a business object can be created, modified, or deleted, a transaction must be started. Changes to the affected business objects will only be made durable when the transaction is committed.

Always remember that when you need to create, change, or delete an object instance, a transaction is required.

Transaction.wrap(function () {
   // Your code here
});

Transaction Types

There are two types of transactions: Implicit and Explicit.

  • Implicit: This is the example provided above. You define the code inside the transaction without explicitly writing any begin or commit commands. If an error occurs, the transaction will automatically roll back, and the changes will be undone.
  • Explicit: In this type of transaction, you are responsible for defining where the transaction begins and ends.
