üìù Exercise: Set and Retrieve Variables

  1. Create a new controller called VarTest.
  2. Create a new ISML template called vartest.isml.
  3. In the template, create a new variable called sessionVar with hardcoded text as the value. Set its scope to session and print the value to the page: 
    <isset name="sessionVar" value="${1}" scope="session"/>
  4. Display the contents of the sessionVar variable. Its value is: ${session.custom.sessionVar}.
  5. Open a web browser and test the controller.
  6. Add similar examples using different scopes (request and page) to the vartest template and display them.
  7. Modify the examples to use boolean and string values (e.g., ${false} and Hello).
  8. Test the JavaScript controller again to see the new variable values.
  9. Increment the variables using the following syntax: ${request.custom.requestVar + 1}.
  10. Note: When creating both Request and Session variables, use request.custom.<name> or session.custom.<name> to access the variable. Page variables, on the other hand, can be accessed directly by their name.

üìù Exercise: Use Local Includes

  1. Study the productTile.isml template.
  2. Notice that the first <isset> tag expects pdict.product in the pipeline dictionary.
  3. Include productTile.isml in your current template.
  4. Create a new controller called ShowProduct with one endpoint where you will retrieve a product using the ProductMgr class.
  5. Send your product to your template.
  6. Open the productfound.isml template.
  7. Create a new variable named product that receives the value of your product being sent from the template: 
    <isset name="product" value="${pdict.myProduct}" scope="request"/>
  8. Test the JavaScript controller with an existing product: <your_storefront_url>/ShowProduct-Start?pid=P0048M.

üìù Exercise: Use Remote Includes

  1. Create two endpoints in your controller named RenderTemplate and TestRemoteInclude.
  2. Create a new ISML template with any dummy text.
  3. In the RenderTemplate endpoint, render this template.
  4. Create a second ISML template and use a remote include to call your RenderTemplate controller endpoint.
  5. In the TestRemoteInclude endpoint, render your second template.
  6. On your storefront site URL, call your TestRemoteInclude endpoint.

üìù Exercise: Use a Decorator

  1. On Visual Studio Code, use the Search function to locate the common/layout/page.isml template. Notice the different areas of the page this decorator defines.
  2. Locate the <isdecorate> tag.
  3. Open the controller you created in the previous exercise and create a new endpoint called TestDecorator.
  4. Create a new ISML template to be rendered by this new endpoint.
  5. Inside your ISML template, add the following: 
    <isdecorate template="common/layout/page">
             ..existing content..
        </isdecorate>
  6. Test the endpoint.
