The pdict variable is a hashmap on which key-object pairs can be loaded on the frontend. However, there are some built-in pdict keys (variables) that provide access to the most commonly used objects, such as session and request. This means that any data you send from the backend to the frontend, whether using res.render or res.setViewData, will be loaded inside the pdict variable on the frontend.

ISML expressions provide access to data using dot notation. For example, you can access a property of the Product object in the pipeline dictionary with: ${pdict.varName}

JavaScript controllers can use alternatives to pdict keys. The table above shows some of them.

In other words, controllers have access to request, response, session, and customer objects just like you have on the frontend using pdict. You just need to use the appropriate import or require statements.
