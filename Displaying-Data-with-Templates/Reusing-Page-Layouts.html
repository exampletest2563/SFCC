Reusing Page Layouts

One very useful feature that ISML offers is the ability to create page layouts and reuse these layouts on any pages. This helps avoid creating pages from scratch every time.

Additionally, it simplifies maintenance. If you need to update something on the page, including SCSS or JavaScript imports, you only need to modify the page layout being used. The changes will automatically be reflected on all pages that use that layout.

To create and use these page layouts, we use a special type of template called a decorator template. You can apply a decorator template using the <isdecorate> tag.

These files are stored in a specific folder: app_storefront_base/cartridge/templates/default/common/layout.

The decorator template uses the <isreplace> tag to specify where the content should be replaced by the decorated content. The following example illustrates a decorator and the area where the content is replaced by whatever template uses the decorator:

A typical use case is decorating the content body with a header and footer. See the app_storefront_base/cartridge/templates/default/common/layout/page.isml file for an example.

Typically, the decorator template uses only one tag, <isreplace>. However, you can use multiple tags. If the decorator template contains multiple <isreplace> tags, the content to be decorated will be included for each <isreplace> tag.

Here is an example where the homepage uses a decorator:

<isdecorate template="common/layout/page" />
